## PROMPT START

You are helping me build **MindBridge**, an offline mental health companion chatbot for students, built for the **Gemma 4 Good Hackathon** on Kaggle (sponsored by Google DeepMind). This project must be complete, functional, and demo-ready.

---

## PROJECT CONTEXT

- **Model:** Gemma 4 E4B running locally via Ollama (`ollama run gemma4:e4b`)
- **Ollama API base URL:** `http://localhost:11434`
- **UI Framework:** Gradio
- **Python version:** 3.10+
- **Platform:** Windows (local machine, CPU only, 16GB RAM)
- **No GPU. No cloud APIs. No internet required at runtime.**
- All data stays local. No external API calls except to localhost Ollama.

## FILE SPECIFICATIONS

### 1. `prompt.py`

Create a file that contains the system prompt string for Gemma 4.

The system prompt must:
- Give the AI the persona of a warm, empathetic student mental health companion named "MindBridge"
- Make it speak in simple, friendly, non-clinical language suitable for 18–25 year old Indian students
- Tell it to NEVER diagnose any mental health condition
- Tell it to NEVER give medical advice
- Tell it to gently suggest professional help or the iCall helpline (9152987821) if the user seems in crisis
- Tell it to offer coping exercises (breathing, grounding, journaling) when appropriate
- Tell it to keep responses concise — 3 to 5 sentences max per reply
- Tell it to always be non-judgmental, warm, and encouraging
- Tell it the user's conversation is private and never shared

Store the prompt as a Python constant called `SYSTEM_PROMPT`.

---

### 2. `ollama_client.py`

Create a Python class `OllamaClient` that:
- Connects to Ollama at `http://localhost:11434`
- Uses the model `gemma4:e4b`
- Has a method `chat(messages: list) -> str` that:
  - Sends a POST request to `/api/chat`
  - Passes the full conversation history as `messages`
  - Sets `stream: False`
  - Returns the assistant's reply as a plain string
- Has a method `is_available() -> bool` that:
  - Pings `http://localhost:11434` 
  - Returns True if Ollama is running, False otherwise
- Handles connection errors gracefully with a clear error message

---

### 3. `mood_tracker.py`

Create a `MoodTracker` class that:
- Stores mood logs in `data/mood_log.json`
- Has a method `log_mood(score: int, note: str)` that:
  - Saves a mood entry with timestamp, score (1–5), and optional note
  - Appends to the JSON file (does not overwrite)
- Has a method `get_recent(n: int) -> list` that returns the last n mood entries
- Has a method `get_average() -> float` that returns the average mood score
- Creates the `data/` directory and JSON file if they don't exist

---

### 4. `exercises.py`

Create a file with a dictionary called `EXERCISES` containing at least 5 coping exercises. Each exercise should have:
- `name`: short title
- `description`: 1-sentence explanation
- `steps`: list of 3–5 step-by-step instructions

Include these exercises:
1. Box breathing (4-4-4-4 technique)
2. 5-4-3-2-1 grounding technique
3. Progressive muscle relaxation
4. Journaling prompt (3 things you're grateful for)
5. Body scan meditation

Also create a function `get_exercise(name: str) -> dict` that returns the exercise by name.
And a function `list_exercises() -> list` that returns all exercise names.

---

### 5. `app.py`

Create the main Gradio application. It must:

**On startup:**
- Check if Ollama is running using `OllamaClient.is_available()`
- If not running, show a clear error message telling the user to start Ollama
- Initialize the conversation history with the system prompt from `prompt.py`

**UI Layout (use `gr.Blocks`):**
- App title: "🧠 MindBridge" with subtitle "Your private mental health companion"
- A mood check-in row at the top:
  - A slider (1–5) labeled "How are you feeling right now?"
  - Emoji labels: 1=😔 2=😟 3=😐 4=🙂 5=😊
  - A "Log Mood" button that saves mood using MoodTracker
- A main chat window (`gr.Chatbot`) showing conversation history
- A text input box for the user message with a "Send" button
- A sidebar or accordion with:
  - "Coping Exercises" section — dropdown to pick an exercise, button to load it
  - "My Mood History" section — shows last 7 mood entries as a simple text list
  - "Journal" section — a text area to write a journal entry with a "Save Entry" button
- A footer with: "MindBridge runs entirely on your device. Your conversations are private."

**Chat behavior:**
- Maintain full conversation history in a Gradio state variable
- On each message, append user message to history, call `OllamaClient.chat()`, append response
- Display the full chat history in the Chatbot component
- Clear the input box after sending
- Show a "Thinking..." placeholder while waiting for Gemma 4 to respond

**Exercise behavior:**
- When user selects an exercise and clicks load, display its steps in the chat window as a formatted message from MindBridge

**Journal behavior:**
- Save journal entries to `data/journal.json` with timestamp
- Show a confirmation message after saving

**Error handling:**
- If Ollama is not running, show: "⚠️ MindBridge cannot connect to Gemma 4. Please make sure Ollama is running by opening a terminal and typing: ollama serve"
- If a chat request fails, show: "I'm having trouble thinking right now. Please try again in a moment."
